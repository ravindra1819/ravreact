Implementation of JavaScript

1. Inline JavaScript

Inline JavaScript is written directly inside the <tag> of an HTML element using attributes such as onclick, onmouseover, etc. It is quick and useful for very small scripts, but not recommended for large projects.

2. Internal JavaScript

Internal JavaScript is written inside a <script> tag, usually at the bottom of the <body>. This keeps JavaScript separate from the HTML elements and is more organized compared to inline scripting.

3. External JavaScript

External JavaScript is stored in a separate file with a .js extension and linked to the HTML using the <script src="..."> tag. This is the most recommended way since it separates structure (HTML), styling (CSS), and behavior (JavaScript) â€” making code reusable and easier to maintain.

Difference Between Inline, Internal, and External JavaScript

Aspect Inline Internal External
Location Inside HTML tag Inside <script> tag in same HTML file In a separate .js file
Ease of Maintenance Difficult Moderate Easy
Reusability Not reusable Limited Highly reusable
Performance Loads with HTML Loads with HTML Can be cached by browser
